# Markdown 快速入门

> 面向团队日常写作：README、操作文档、评审说明等。

## 常用语法

- 标题：`# 一级标题` ~ `###### 六级标题`。
- 段落与换行：空行分段，行末加两个空格或空行换行。
- 强调：`*斜体*`，`**加粗**`，`~~删除线~~`。
- 列表：`-`/`*`/`+` 创建无序列表，`1.` 创建有序列表，缩进两空格生成嵌套层级。
- 链接与图片：`[文本](url)`，`![描述](path/to/img)`。
- 代码：行内 `\`code\``，多行使用三反引号：

  ```
  ```bash
  ls -la
  ```
  ```

- 引用：`>` 开头，支持嵌套。
- 表格：使用 `|` 分隔列，第二行用 `---` 或 `:---:` 控制对齐。

## 写作建议

- 标点与空格：中英文混排时在英文两侧留一个空格，例如“使用 Markdown 编写说明”。
- 结构清晰：用二级/三级标题分隔章节，并配合列表突出关键步骤。
- 贴图规范：图片放入 `media/` 或子目录，文件名简短，如 `workflow-overview.png`。
- 代码高亮：在三反引号后添加语言标识（如 `bash`、`python`）便于阅读。
- 链接检查：内部引用使用相对路径，避免失效链接。

## 进阶用法

- 任务列表：`- [ ] 待办`、`- [x] 完成`。
- 脚注：`文字[^1]` 与文末 `[^1]: 注释内容`。
- 折叠块（GitHub 支持）：

  ```html
  <details>
    <summary>点击展开</summary>
    详细内容
  </details>
  ```

## 提交前自查

- 是否遵循团队术语与命名规范。
- 图片路径、表格对齐是否正确。
- 说明步骤是否可复现，必要时补充命令或示例输出。
